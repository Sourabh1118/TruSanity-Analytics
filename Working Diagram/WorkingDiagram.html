Netra Analytics Data Engine

A visual representation of how data flows from your customers' websites down to the exact databases, and where the UI dashboards render that data.

1. Data Sources (The Wild)

Web / Browser SDK

Users visit your customer's website. The `@netra/sdk-js` tracks every click, pageview, and scroll automatically.

Functional - Triggers HTTP

E-Commerce Plugins

Shopify or Magento fires a backend Webhook when an "Order Paid" event occurs securely on the server.

Functional - Server-to-Server

Mobile Apps

React Native/iOS apps batch up data and send it when the phone restores internet connection.

Functional - Offline Queues

2. Fastify Edge API (apps/api)

The high-throughput traffic cop. It catches millions of incoming JSON blocks per second without crashing.

Active

Redis Cache

Verifies if the incoming Project ID is a valid paying customer. Also rate-limits abusive IPs instantly.

Active

Apache Kafka Queue

Throws the valid raw payload onto the `netra.events` queue and tells the browser "HTTP 202 Accepted" so the app doesn't freeze.

3. Background Worker & Data Warehouse

Daemon Running

Consumer Node.js

A background script (`consumer.ts`) quietly reading from Kafka in chunks of 500. It cleans up the timestamps and converts the data into massive JSON blocks.

Holding Data

ClickHouse Database

The absolute beast. It receives the 500-event blocks and writes them to the solid-state drives in highly compressed columns. Built for math and speed.

4. Next.js SaaS Web Interface (apps/web)

The beautiful front-end that you (Super Admin) and your SaaS customers interact with via web browsers at `localhost:3000`.

Active

PostgreSQL + Drizzle

Verifies your password. Next.js Middleware checks if you are a `SUPER_ADMIN` or a standard `user` and locks down the routes to keep SaaS data secure.
